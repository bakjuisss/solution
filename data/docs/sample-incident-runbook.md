# PCGuard 장애 대응 Runbook

## 장애 등급 기준

| 등급 | 기준 | 대응 |
|------|------|------|
| 낮음 | 단일 PC, 업무 영향 없음 | 1차 엔지니어 원격 지원 |
| 보통 | 부서 단위 영향 | 4시간 내 조치, 기술지원팀 공유 |
| 높음 | 전사 에이전트 다수 미수신 | 1시간 내 대응, 제품팀 에스컬레이션 |
| 긴급 | 관리 서버 다운, 보안 정책 전체 미적용 | 즉시 전화 연락, 24시간 대응 |

## INC-001: 에이전트 서버 연결 실패

### 증상
- 에이전트 상태가 "오프라인" 또는 "연결 실패"
- 관리 콘솔에 에이전트가 표시되지 않음

### 가능 원인
1. 방화벽에서 8443, 8080 포트 차단
2. 관리 서버 주소 설정 오류
3. 에이전트 서비스 중지
4. 네트워크 프록시/SSL 검사 간섭

### 확인 순서
1. 에이전트 PC에서 `telnet 관리서버IP 8443` 연결 테스트
2. 레지스트리 `HKLM\Software\Plantynet\PCGuard\Server` 값 확인
3. Windows 서비스에서 PCGuard Agent 서비스 상태 확인
4. 관리 서버 방화벽 인바운드 규칙 확인

### 조치
1. 방화벽에 8443(TCP), 8080(TCP) 허용 규칙 추가
2. 서버 주소가 올바르면 에이전트 재시작 (`net stop PCGuardAgent` → `net start PCGuardAgent`)
3. 주소 오류 시 GPO 또는 수동으로 올바른 서버 주소 재배포
4. SSL 검사 환경이면 관리 서버 URL 예외 등록

### 에스컬레이션
- 30분 내 조치 불가 시 기술지원팀(내선 1234) 에스컬레이션
- 전사 50% 이상 에이전트 미수신 시 긴급 등급으로 상향

### 고객 안내
"연결 문제는 주로 방화벽 포트 또는 서버 주소 설정에서 발생합니다. 포트 개방과 에이전트 재시작을 진행 중이며, 30분 내 결과를 안내드리겠습니다."

## INC-002: 라이선스 오류

### 증상
- "라이선스가 유효하지 않습니다" 메시지
- 등록 에이전트 수 0 표시

### 가능 원인
1. 라이선스 키 만료 또는 에이전트 수 초과
2. 서버 시스템 시간 불일치 (NTP 미동기화)
3. 라이선스 파일 손상

### 확인 순서
1. 웹 콘솔 > 설정 > 라이선스 관리에서 만료일·등록 수 확인
2. 서버 `date /t` 및 NTP 동기화 상태 확인
3. 라이선스 키 재업로드 시도

### 조치
1. NTP 동기화 후 관리 서버 재시작
2. 유효한 라이선스 키 재등록
3. 에이전트 수 초과 시 미사용 에이전트 정리 또는 라이선스 증설 요청

### 에스컬레이션
- 라이선스 증설 필요 시 영업/계약 담당자 연결
- 키 자체 오류 시 제품팀 확인 요청

## 민원 응대 가이드

### CMP-001: 성능 저하 민원

**고객 응대 멘트:**
"PCGuard 설치 후 체감 속도 저하를 호소하신 점 이해합니다. 에이전트는 시스템 리소스를 최소한으로 사용하도록 설계되어 있으며, 실시간 검사 예외 설정이나 스캔 정책 조정으로 개선할 수 있습니다. 원격 점검 후 최적 설정을 안내드리겠습니다."

**내부 조치:**
1. 해당 PC의 CPU/메모리 사용량 확인
2. 실시간 검사 예외 대상(개발 도구 등) 등록 여부 확인
3. 정책에서 불필요한 실시간 검사 항목 완화 검토

### CMP-002: 제거 방법 문의

**고객 응대 멘트:**
"제거 절차를 안내드리겠습니다. 제어판 또는 제공된 제거 도구로 에이전트를 삭제할 수 있으며, 제거 후에는 보안 정책이 적용되지 않으므로 관리자 승인 후 진행을 권장합니다."

**내부 조치:**
1. 제거 사유 및 관리자 승인 여부 확인
2. 제거 도구(`PCGuardUninstall.exe /silent`) 경로 안내
3. 제거 후 GPO 재배포 시 재설치 여부와 담당자 공유
